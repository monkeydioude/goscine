## schema memoire

main -> sliceA(0x000) := [1(0x0001), 2(0x0002), 3(0x0003)]
main -> step1(sliceA=0x1000)
step1 -> AppendSlicePtr(&sliceA=0x1000, [4(0x0004), 5(0x0005), 6(0x0006)])

step1 -> ModifySlice(sliceA=0x2000)
ModifySlice-> s[0(0x011)] = 999
step1 -> sliceA == [999(0x011), 2(0x012), 3(0x013), 4(0x014), 5(0x015), 6(0x016)]
main-> sliceA == [1(0x0001), 2(0x0002), 3(0x0003)]

## explication

main -> sliceA(0x000) := [1(0x0001), 2(0x0002), 3(0x0003)]
=> on créé une nouvelle slice (`sliceA`). L'adresse du debut de la `sliceA` est 0x000, elle a 3 elements, chaque element avec son adresse 0x0001, 0x0002, 0x0003 et sa capacité est 3.

main -> step1(sliceA=0x1000)
=> la `sliceA` est passée **par copie**. Donc la `sliceA` a une nouvelle adresse (0x1000), mais l'adresse de chaque element de la `sliceA` reste le meme (0x0001, 0x0002, 0x0003)

step1 -> AppendSlicePtr(&sliceA=0x1000, [4(0x0004), 5(0x0005), 6(0x0006)])
=> on ajoute des elements à la `sliceA`. La `sliceA` est passée par reference, donc son adresse ne change pas (0x1000). Dans cette fonction, on ajoute des elements a la `sliceA`. Cependant, `sliceA` a une capacité de 3, et on essaye d'ajouter 3 elements. Donc la capacité est insuffisante. Une nouvelle slice remplace donc `sliceA` (0x2000), avec une capacité de 6, et les elements 4, 5, 6 sont ajoutes a la fin de `sliceA`. L'adresse de chaque element change egalement de `sliceA`.

## Qu'est ce qu'une slice ([]int) ?

Une slice est en fait un tableau dynamique. C'est une structure de données qui permet de stocker une collection d'éléments de même type.
Une slice est en fait une structure compose de 3 champs:

- pointer: Pointeur vers le premier élément du tableau dynamique.
- length: Nombre d'éléments dans la slice.
- capacity: Nombre total d'éléments du tableau dynamique.

```go
type SliceHeader struct {
	pointer unsafe.Pointer
	length  int
	capacity  int
}
```

## comment déclarer une slice ?

var sliceA []int
sliceA := []int{1, 2, 3}
