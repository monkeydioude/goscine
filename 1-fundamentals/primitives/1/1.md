main -> sliceA(0x000) := [1(0x0001), 2(0x0002), 3(0x0003)]
main -> step1(sliceA=0x000A)
step1 -> AppendSlicePtr(&sliceA=0x000A, [4(0x0004), 5(0x0005), 6(0x0006)])
AppendSlicePtr -> \*[1(0x011), 2(0x012), 3(0x013)] + [4(0x014), 5(0x015), 6(0x016)]

## explication

main -> sliceA(0x000) := [1(0x0001), 2(0x0002), 3(0x0003)]
=> on créé une nouvelle slice (`sliceA`). L'adresse du debut de la `sliceA` est 0x000, elle a 3 elements, chaque element avec son adresse 0x0001, 0x0002, 0x0003 et sa capacité est 3.

main -> step1(sliceA=0x000A)
=> la `sliceA` est passée **par copie**. Donc dans `step1`, la `sliceA` a une nouvelle adresse (0x000A). Par contre, l'adresse de chaque element de la `sliceA` reste le meme (0x0001, 0x0002, 0x0003)

step1 -> AppendSlicePtr(&sliceA=0x000A, [4(0x0004), 5(0x0005), 6(0x0006)])
=> on ajoute des elements à la `sliceA`. La `sliceA` est passée par reference, donc son adresse ne change pas (0x000A). Dans cette fonction, on ajoute des elements a la `sliceA`. Cependant, `sliceA` a une capacité de 3, et on essaye d'ajouter 3 elements. Donc la capacité est insuffisante. Une nouvelle slice remplace donc `sliceA` (mais toujours a l'adresse 0x000A) dans la fonction `step1`, avec une capacité de 6, et les elements 4, 5, 6 sont ajoutes a la fin de `sliceA`. L'adresse de chaque element change egalement de `sliceA`.

step1 -> ModifySlice(sliceA=0x000B)
=> on modifie la `sliceA`. La `sliceA` est passée par copie, donc son adresse dans la fonction ModifySlice change (0x000B). Par contre, l'adresse de chaque element de la `sliceA` reste le meme (0x011, 0x012, 0x013, 0x014, 0x015, 0x016). Dans cette fonction, on modifie la valeur du premier element de la `sliceA`. Les adresses des elements de `sliceA` ne bougent pas.

step1 -> sliceA == [999(0x011), 2(0x012), 3(0x013), 4(0x014), 5(0x015), 6(0x016)]
=> On est de retour dans step1. L'adresse de `sliceA` est toujours 0x000A. L'adresse de chaque element n'a pas changé, toujours 0x011, 0x012, 0x013, 0x014, 0x015, 0x016. Par contre la **valeur** du premier element est maintenant **999**.

main-> sliceA == [1(0x0001), 2(0x0002), 3(0x0003)]
=> On est de retour dans main. L'adresse de `sliceA` est toujours 0x000. L'adresse de chaque element n'a pas changé, toujours 0x001, 0x002, 0x003. Les valeurs sont 1, 2, 3. Les elements 4, 5, 6 n'ont pas été ajoutés à la `sliceA` de main (adresse differente et une nouvelle slice a ete créée a cause du append dans `AppendSlicePtr`)

## Qu'est ce qu'une slice ([]int) ?

Une slice est en fait un tableau dynamique. C'est une structure de données qui permet de stocker une collection d'éléments de même type.
Une slice est en fait une structure compose de 3 champs:

- pointer: Pointeur vers le premier élément du tableau dynamique.
- length: Nombre d'éléments dans la slice.
- capacity: Nombre total d'éléments du tableau dynamique.

```go
type SliceHeader struct {
	pointer unsafe.Pointer
	length  int
	capacity  int
}
```

## Объяснение

main -> sliceA(0x000) := [1(0x0001), 2(0x0002), 3(0x0003)]
=> мы создаем новый слайс (`sliceA`). Адрес начала `sliceA` — 0x000, в нем 3 элемента, каждый элемент имеет свой адрес 0x0001, 0x0002, 0x0003, а его вместимость (capacity) равна 3.

main -> step1(sliceA=0x000A)
=> `sliceA` передается **по копии**. Поэтому в `step1` у `sliceA` новый адрес (0x000A). Однако адрес каждого элемента `sliceA` остается прежним (0x0001, 0x0002, 0x0003).

step1 -> AppendSlicePtr(&sliceA=0x000A, [4(0x0004), 5(0x0005), 6(0x0006)])
=> мы добавляем элементы в `sliceA`. `sliceA` передается по ссылке, поэтому его адрес не меняется (0x000A). В этой функции мы добавляем элементы в `sliceA`. Однако вместимость `sliceA` равна 3, а мы пытаемся добавить 3 элемента. Следовательно, вместимости недостаточно. Поэтому новый слайс заменяет `sliceA` (но все так же по адресу 0x000A) в функции `step1`, с вместимостью 6, и элементы 4, 5, 6 добавляются в конец `sliceA`. Адрес каждого элемента `sliceA` также меняется.

step1 -> ModifySlice(sliceA=0x000B)
=> мы изменяем `sliceA`. `sliceA` передается по копии, поэтому его адрес в функции ModifySlice меняется (0x000B). Однако адрес каждого элемента `sliceA` остается прежним (0x011, 0x012, 0x013, 0x014, 0x015, 0x016). В этой функции мы изменяем значение первого элемента `sliceA`. Адреса элементов `sliceA` не перемещаются.

step1 -> sliceA == [999(0x011), 2(0x012), 3(0x013), 4(0x014), 5(0x015), 6(0x016)]
=> Мы возвращаемся в step1. Адрес `sliceA` по-прежнему 0x000A. Адрес каждого элемента не изменился, все те же 0x011, 0x012, 0x013, 0x014, 0x015, 0x016. Однако **значение** первого элемента теперь равно **999**.

main-> sliceA == [1(0x0001), 2(0x0002), 3(0x0003)]
=> Мы возвращаемся в main. Адрес `sliceA` по-прежнему 0x000. Адрес каждого элемента не изменился, все те же 0x001, 0x002, 0x003. Значения — 1, 2, 3. Элементы 4, 5, 6 не были добавлены в `sliceA` функции main (другой адрес, и новый слайс был создан из-за append в `AppendSlicePtr`).

## Что такое слайс ([]int)?

Слайс — это по сути динамический массив. Это структура данных, которая позволяет хранить коллекцию элементов одного типа.
На самом деле слайс представляет собой структуру, состоящую из 3 полей:

- pointer: Указатель на первый элемент динамического массива.
- length: Количество элементов в слайсе.
- capacity: Общее количество элементов в динамическом массиве.

```go
type SliceHeader struct {
	pointer unsafe.Pointer
	length  int
	capacity  int
}
```

## Explanation

main -> sliceA(0x000) := [1(0x0001), 2(0x0002), 3(0x0003)]
=> we create a new slice (`sliceA`). The starting address of `sliceA` is 0x000, it has 3 elements, each element with its own address 0x0001, 0x0002, 0x0003, and its capacity is 3.

main -> step1(sliceA=0x000A)
=> `sliceA` is passed **by copy** (by value). So in `step1`, `sliceA` has a new address (0x000A). However, the address of each element of `sliceA` remains the same (0x0001, 0x0002, 0x0003).

step1 -> AppendSlicePtr(&sliceA=0x000A, [4(0x0004), 5(0x0005), 6(0x0006)])
=> we add elements to `sliceA`. `sliceA` is passed by reference (pointer), so its address does not change (0x000A). In this function, we add elements to `sliceA`. However, `sliceA` has a capacity of 3, and we are trying to add 3 elements. Therefore, the capacity is insufficient. A new slice backing array is allocated, and the slice header (still at address 0x000A) is updated in the `step1` function, now with a capacity of 6, and the elements 4, 5, 6 are added to the end of `sliceA`. The memory address of each element of `sliceA` also changes.

step1 -> ModifySlice(sliceA=0x000B)
=> we modify `sliceA`. `sliceA` is passed by copy, so its address in the ModifySlice function changes (0x000B). However, the address of each element of `sliceA` remains the same (0x011, 0x012, 0x013, 0x014, 0x015, 0x016). In this function, we modify the value of the first element of `sliceA`. The addresses of the elements of `sliceA` do not move.

step1 -> sliceA == [999(0x011), 2(0x012), 3(0x013), 4(0x014), 5(0x015), 6(0x016)]
=> We are back in step1. The address of `sliceA` is still 0x000A. The address of each element has not changed, still 0x011, 0x012, 0x013, 0x014, 0x015, 0x016. However, the **value** of the first element is now **999**.

main-> sliceA == [1(0x0001), 2(0x0002), 3(0x0003)]
=> We are back in main. The address of `sliceA` is still 0x000. The address of each element has not changed, still 0x001, 0x002, 0x003. The values are 1, 2, 3. The elements 4, 5, 6 were not added to the `sliceA` in main (different address, and a new slice backing array was created due to the append in `AppendSlicePtr`).

## What is a slice ([]int)?

A slice is essentially a dynamic array. It is a data structure that allows you to store a collection of elements of the same type.
Under the hood, a slice is a structure composed of 3 fields:

- pointer: A pointer to the first element of the dynamic array.
- length: The number of elements in the slice.
- capacity: The total number of elements the dynamic array can hold before needing reallocation.

```go
type SliceHeader struct {
	pointer unsafe.Pointer
	length  int
	capacity  int
}
```

## 解説

main -> sliceA(0x000) := [1(0x0001), 2(0x0002), 3(0x0003)]
=> 新しいスライス (`sliceA`) を作成します。`sliceA` の開始アドレスは 0x000 で、3つの要素を持ち、各要素は自身のアドレス 0x0001, 0x0002, 0x0003 を持ちます。また、そのキャパシティ（容量）は 3 です。

main -> step1(sliceA=0x000A)
=> `sliceA` は**値渡し（コピー）**されます。そのため、`step1` 内では、`sliceA` は新しいアドレス (0x000A) を持ちます。しかし、`sliceA` の各要素のアドレスは同じままです (0x0001, 0x0002, 0x0003)。

step1 -> AppendSlicePtr(&sliceA=0x000A, [4(0x0004), 5(0x0005), 6(0x0006)])
=> `sliceA` に要素を追加します。`sliceA` は参照（ポインタ）で渡されるため、そのアドレスは変わりません (0x000A)。この関数内で `sliceA` に要素を追加します。しかし、`sliceA` のキャパシティは 3 であり、3つの要素を追加しようとしています。したがって、キャパシティが不足します。新しい基礎配列がメモリに割り当てられ、`step1` 関数内のスライスヘッダ（アドレス 0x000A のまま）が更新されます。キャパシティは 6 になり、要素 4, 5, 6 が `sliceA` の末尾に追加されます。`sliceA` の各要素のメモリアドレスも変更されます。

step1 -> ModifySlice(sliceA=0x000B)
=> `sliceA` を変更します。`sliceA` はコピーで渡されるため、ModifySlice 関数内でのアドレスは変わります (0x000B)。しかし、`sliceA` の各要素のアドレスは同じままです (0x011, 0x012, 0x013, 0x014, 0x015, 0x016)。この関数内で、`sliceA` の最初の要素の値を変更します。`sliceA` の要素のアドレスは移動しません。

step1 -> sliceA == [999(0x011), 2(0x012), 3(0x013), 4(0x014), 5(0x015), 6(0x016)]
=> step1 に戻ります。`sliceA` のアドレスは依然として 0x000A です。各要素のアドレスは変更されておらず、0x011, 0x012, 0x013, 0x014, 0x015, 0x016 のままです。ただし、最初の要素の**値**は **999** になっています。

main-> sliceA == [1(0x0001), 2(0x0002), 3(0x0003)]
=> main に戻ります。`sliceA` のアドレスは依然として 0x000 です。各要素のアドレスは変更されておらず、0x001, 0x002, 0x003 のままです。値は 1, 2, 3 です。要素 4, 5, 6 は main の `sliceA` には追加されていません（異なるアドレスであり、`AppendSlicePtr` での append により新しい基礎配列が作成されたため）。

## スライス ([]int) とは何か？

スライスは本質的に動的配列です。同じ型の要素のコレクションを格納できるデータ構造です。
内部的には、スライスは以下の3つのフィールドで構成される構造体です。

- pointer: 動的配列の最初の要素へのポインタ。
- length: スライス内の要素数。
- capacity: 動的配列が再割り当てを必要とするまでに保持できる要素の総数。

```go
type SliceHeader struct {
	pointer unsafe.Pointer
	length  int
	capacity  int
}
```
